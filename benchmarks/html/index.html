About these benchmarks:

Each benchmark runs multiple iterations to get stable measurements. Results show operations per second (higher is better) and relative performance comparison.

Note: Browser JIT warmup can affect initial results. Each benchmark includes warmup iterations.

Important: These benchmarks compare a pure JavaScript/TypeScript implementation of the new Streams API against the browser's fully optimized native implementation of Web Streams. The Web Streams API is implemented in C++ and has been heavily optimized over years of development. Results should be interpreted with this context in mind - competitive performance from a JS implementation is a positive signal, and a native implementation of the new API would likely perform significantly better.
